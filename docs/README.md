## 阅读顺序

首次参与项目时，建议依次阅读：

1. [总体架构](architecture.md)，理解模块职责和依赖禁区。
2. [配置设计](configuration.md)与[运行模型](runtime.md)，理解任务如何从文件变成受监管进程。
3. 根据负责模块阅读[可观测性](observability.md)或[平台适配](platform.md)。
4. 开始实现前确认[测试策略](testing.md)和[实现路线图](roadmap.md)中的当前阶段。

## 文档维护规则

- 设计文档使用中文，公共术语首次出现时可附英文名。
- 尚未实现的内容必须标为“设计”或“计划”，不能写成已具备能力。
- 协议、配置和持久化格式发生不兼容变更时，应补充架构决策记录（ADR）。
- 每个里程碑完成后同步更新路线图、README 的当前状态和对应模块文档。

ADR 存放于 `docs/adr/`，文件名使用 `NNNN-short-title.md`，并记录背景、决策、替代方案和后果。
